Attributes modify the behaviour or appearance of HTML elements. They are written inside the opening tag as name-value pairs.

## Attribute Syntax

```html
<tagname attribute="value">content</tagname>
```

- A space separates the tag name from the first attribute
- Multiple attributes are separated by spaces
- Values should be wrapped in double quotes (single quotes also work)

```html
<a href="https://example.com" target="_blank" title="Visit Example">
  Click here
</a>
```

## Global Attributes

These attributes work on **every** HTML element.

| Attribute | Purpose |
|---|---|
| `class` | Assigns one or more CSS class names for styling |
| `id` | Provides a unique identifier for the element (use once per page) |
| `style` | Adds inline CSS styles directly on the element |
| `lang` | Specifies the language of the element's content |
| `hidden` | Hides the element from rendering |
| `tabindex` | Controls keyboard tab order |
| `title` | Shows a tooltip on hover |

```html
<div class="card featured" id="card-1" style="background: #f0f0f0;">
  <p lang="fr">Bonjour le monde!</p>
</div>

<button title="Save your changes">Save</button>
<p hidden>This paragraph is hidden.</p>
```

## Element-Specific Attributes

Some attributes only apply to specific elements.

### Links (`<a>`)

```html
<a href="/page" target="_blank" rel="noopener noreferrer">Link</a>
```

- `href` ‚Äî destination URL
- `target="_blank"` ‚Äî opens in a new tab
- `rel` ‚Äî describes the relationship (use `noopener noreferrer` with `_blank` for security)

### Images (`<img>`)

```html
<img src="photo.jpg" alt="A sunny beach at sunset" width="800" height="600">
```

- `src` ‚Äî image path or URL
- `alt` ‚Äî descriptive text for accessibility and fallback (always include)
- `width` / `height` ‚Äî dimensions in pixels (helps prevent layout shift)

### Forms (`<input>`, `<button>`, etc.)

```html
<input type="text" name="username" placeholder="Enter your name" required>
<button type="submit" disabled>Submit</button>
```

- `type` ‚Äî input type (text, email, password, number, etc.)
- `name` ‚Äî key sent to the server
- `placeholder` ‚Äî hint text inside the input
- `required` ‚Äî prevents submission if empty
- `disabled` ‚Äî makes the element unclickable
- `value` ‚Äî pre-filled or submitted value

## Boolean Attributes

Some attributes don't need a value. Their presence alone means `true`, and omitting them means `false`.

```html
<!-- These mean the same thing -->
<input required>
<input required="required">
<input required="true">

<!-- Without the attribute -->
<input>  <!-- not required -->
```

Common boolean attributes: `disabled`, `readonly`, `checked`, `selected`, `hidden`, `required`.

## Data Attributes

Custom attributes prefixed with `data-` let you store extra information on elements, accessible via JavaScript.

```html
<button data-user-id="42" data-role="admin">Edit Profile</button>
```

```js
const btn = document.querySelector('button');
console.log(btn.dataset.userId); // "42"
console.log(btn.dataset.role);   // "admin"
```

## Putting It Together

```html
<article class="post" id="post-3" lang="en">
  <h2 class="post-title">My Vacation</h2>
  <img src="beach.jpg" alt="Tropical beach at sunset" width="600" height="400">
  <p>We had an amazing time exploring the <span lang="es">playa</span>.</p>
  <a href="/posts/3" class="read-more" data-category="travel">Read more &rarr;</a>
</article>
```

Attributes give you fine-grained control over how elements behave and appear. Mastering them is key to writing expressive, accessible HTML.
